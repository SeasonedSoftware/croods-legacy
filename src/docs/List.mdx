# List

The List component provides a way to fetch a list from an api, using the response as needed, fetching or set a resource and show or use the response data.

## API

| Property      | Type          | Required | Default                  | Description                                                                                                |
| ------------- | ------------- | -------- | ------------------------ | ---------------------------------------------------------------------------------------------------------- |
| name          | String        | true     | -                        | It defines which reducer will be used and, if no path have been provider, will set path to baseUrl/name.   |
| render        | Func          | true     | -                        | A function returning a React Node. Ex: ({ list }, props) -> < JSX />                                       |
| renderError   | Func          | false    | Provider's renderError   | A function returning a React Node that will render when occurs an error. Ex: (error = {}) -> < JSX />      |
| renderLoading | Func          | false    | Provider's renderLoading | A function returning a React Node that will be rendered while wait the server response. Ex: () -> < JSX /> |  |
| parentId      | String/Number | false    | -                        | Parent element's Id, for more specificity in changes.                                                      |
| path          | String        | false    | `/${name}`               | Ex: /foo/bar-\_12?&=                                                                                       |
| disableCache  | Bool          | false    | false                    | Disable chache. When true, will force Croods to make a new API request each time the component renders.    |

## How to use

Don't forget to add a reducer with the same name used here in your rootReducer, with **createReducer**

```javascript
import { List } from 'croods'

export default (
  <List
    name="pets"
    // if path is different than /pets, you can set a custom path like so:
    // path='/customPath'
    render={list => <Component list={list} />}
  />
)
```

### render

The render function is used to receive the api response in the form of an array, **list** , and
pass it to render in any component you like.

### renderError

The renderError function is fired when the server response has a 4xx or 5xx status.
It receives the `error` parameter containing the error message.

### renderLoading

The renderLoading function is fired while there is no response from server.
